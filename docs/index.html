프론트엔드 개발자로서의 성장 과정 기록

👾 프로그램이란?

프로그램은 컴퓨터에게 특정 작업을 수행하도록 지시하는 '코드 덩어리'! 명령어와 데이터의 집합이다. 프로그래머로서 프로그램을 개발할 때 어떻게 메모리를 가장 효율적으로 사용하며 개발할 수 있을지를 고민해 봐야한다고 생각된다.

🌐 Web 기초 개념

브라우저 주소창에 구글을 입력하면 어떤일이 일어날까?

브라우저 주소창에 도메인 이름을 입력하면 DNS(Domain Name System) 조회를 통해 해당 도메인의 IP 주소를 얻고, 받은 IP 주소를 사용하여 해당 서버에 연결한다. 브라우저는 HTTP 프로토콜을 사용하여 서버에 요청을 보내고 (일반적으로 GET 요청), 서버는 해당 요청에 대한 응답을 생성하여 브라우저에게 전달하고, 브라우저는 응답을 처리하여 사용자에게 웹 페이지를 표시한다.

❓그럼 DNS(Domain Name System)이란 무엇일까?

❗️DNS는 인터넷에서 도메인 이름과 IP 주소 간의 변환을 담당하는 시스템! DNS는 사용자가 이해하기 쉬운 도메인 이름 (예: www.google.com)을 컴퓨터가 이해할 수 있는 IP 주소로 매핑하는 역할을 한다.

브라우저 렌더링 원리

브라우저 렌더링이란 HTML,CSS, Javascript 등의 웹페이지 자원을 브라우저가 화면에 그리는 과정이다.

  • DOM 트리 생성: HTML 태그를 노드로 변환하고, 노드간의 계층 관계를 형성
  • CSSOM 생성: CSS 파일을 파싱하여 스타일 정보를 트리로 구성
  • 렌더 트리 생성: DOM과 CSSOM을 결합하여 실제 화면에 표시될 요소만 선택
  • 레이아웃: 각 요소의 크기와 위치 계산
  • 페인팅: 요소의 배경, 테두리, 글자 등을 화면에 그림

🔧 개발 개념

객체지향 프로그래밍 (OOP)

객체지향 프로그래밍이란, 프로그래밍 할때 서로 밀접하게 연관되어 있는 요소들을 객체로 구성해 나가면서 객체끼리 서로 호환 가능하게 하는 것이다.

주요 개념들:

클래스: 객체를 생성하기 위한 템플릿
객체: 클래스의 인스턴스
캡슐화: 데이터와 메서드를 하나의 단위로 묶는 것
상속: 기존 클래스로부터 새로운 클래스 생성

프로세스와 스레드

프로세스: 운영체제에서 실행 중인 프로그램
스레드: 프로세스 내에서 실행되는 작업의 단위

⚡ 현대적 개발 방식

CSR vs SSR vs Next.js

CSR (Client-Side Rendering): 클라이언트에서 렌더링하는 방식

SSR (Server-Side Rendering): 서버에서 렌더링하는 방식

Next.js: CSR과 SSR을 모두 지원하는 React 프레임워크

REST API

서버와 클라이언트 간의 통신을 위한 표준

GET: 리소스 조회
POST: 리소스 생성
PUT: 리소스 업데이트
DELETE: 리소스 삭제

CORS (Cross-Origin Resource Sharing)

웹 브라우저에서 다른 출처의 리소스에 접근할 수 있도록 허용하는 보안 메커니즘

🚀 DevOps & 협업

CI/CD (Continuous Integration/Deployment)

코드를 지속적으로 통합하고 배포하는 개발 방식

MSA (Microservices Architecture)

큰 애플리케이션을 작은 독립적인 서비스들로 분할하는 아키텍처

테스트 코드의 중요성

버그 발생률을 줄이고 유지보수를 효율적으로 하기 위해 테스트 코드를 작성하는 것이 중요하다.
